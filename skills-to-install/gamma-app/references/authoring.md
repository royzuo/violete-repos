# Gamma Authoring

Use this reference only when you need help shaping the markdown before generation.

## Inputs to prefer

- Best case: the user already provides markdown with `---` card breaks.
- Good case: the user provides an outline or source notes.
- Fallback: the user provides only a topic and goal, so you research first.

## Card-writing rules

- Treat each `---` block as one card.
- Keep one main point per card.
- Prefer 3 to 6 bullets over dense paragraphs.
- Use headings that say what the card is doing, not vague labels.
- Add image cues only when they materially improve the output.
- Preserve explicit user structure unless the user asks for a rewrite.

## Narrative patterns

- Briefing: situation, evidence, implications, next steps.
- Pitch: problem, stakes, solution, proof, ask.
- Educational: hook, mechanism, example, takeaway.
- Report-out: headline, supporting data, recommendation, risks.

## Audience adaptation

- Executives: compress detail, lead with decisions and tradeoffs.
- Customers: emphasize outcomes, proof, and clarity.
- Internal teams: include process, dependencies, and risks.
- General audience: reduce jargon and add context before conclusions.

## When converting existing markdown

- Preserve existing `---` boundaries whenever possible.
- Clean up overlong bullets before generation.
- Remove duplicated headings or repeated setup text.
- Do not add filler cards just to reach a slide count.
